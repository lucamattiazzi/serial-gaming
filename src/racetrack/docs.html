Racetrack — API del bot

La corsa vettoriale su carta a quadretti: pura fisica, pura pianificazione. Chi frena troppo perde, chi non frena si schianta.

Come funziona

Sei un punto su una griglia con una velocità vettoriale. A ogni turno scegli un'accelerazione (ax, ay) con componenti in {-1, 0, 1}: la velocità diventa v' = v + a e la posizione p' = p + v'. Le mosse dei due giocatori sono simultanee (le auto sono fantasmi: niente collisioni tra loro). Vince chi taglia per primo il traguardo. Il segmento tra la posizione vecchia e quella nuova non deve mai toccare un muro: uscire di pista = sconfitta immediata. Dopo 200 turni la gara finisce in pareggio. Il tracciato è sorteggiato a ogni partita tra quelli disponibili.

La busta comune

Ogni messaggio che ricevi contiene anche due campi di busta, omessi negli esempi di questa pagina per brevità: game (qui "racetrack") e match, un UUID che identifica la singola partita e resta identico dal primo all'ultimo messaggio. All'inizio di ogni partita arriva inoltre un annuncio con la sola busta e tutti i campi di gioco a null: ignoralo (la tua rispondi ritorna None). È grazie alla busta che il main.py «router» installato dal Laboratorio smista ogni messaggio al modulo del gioco giusto (bot_<gioco>.py) — è ciò che rende possibile la modalità torneo.

{"game": "racetrack", "match": "3f2b1c4e-…", …campi del gioco…}

Messaggio in ingresso: il turno

{"track": ["################################",
           "#..............................#",
           "#.S............................#", ...],
 "you": {"x": 2, "y": 2, "vx": 1, "vy": 0},
 "opp": {"x": 2, "y": 4, "vx": 0, "vy": 1},
 "turn": 3,
 "winner": null}
Campo Significato
track Il tracciato, una stringa per riga (riga 0 in alto): # muro, . pista, S partenze, F traguardo (le celle S e F sono pista a tutti gli effetti).
you / opp Posizione e velocità correnti. y cresce verso il basso.

La tua risposta

{"move": [1, 0]}
  • [ax, ay] con ciascuna componente in {-1, 0, 1}. Qualsiasi altro valore = sconfitta immediata.
  • Tempo massimo: 2 secondi a turno, poi perdi.
  • Se il segmento del tuo spostamento tocca un muro o esce dalla griglia, ti schianti: sconfitta (doppio schianto nello stesso turno = pareggio).
  • Se il segmento attraversa una cella F, hai finito: vittoria (arrivo simultaneo = pareggio).
  • A fine gara arriva {"track": null, ..., "winner": "you" | "opp" | "TIE"}: non rispondere.

Esempio completo

Un bot prudente: accelera verso il traguardo solo se poi riesce ancora a frenare.

def cell(track, x, y):
    if y < 0 or y >= len(track) or x < 0 or x >= len(track[0]):
        return "#"
    return track[y][x]


def segment_ok(track, x0, y0, x1, y1):
    steps = max(abs(x1 - x0), abs(y1 - y0)) * 4 + 1
    for i in range(1, steps + 1):
        x = round(x0 + (x1 - x0) * i / steps)
        y = round(y0 + (y1 - y0) * i / steps)
        c = cell(track, x, y)
        if c == "#":
            return False
        if c == "F":
            return True
    return True


def can_stop(track, x, y, vx, vy):
    while vx or vy:
        vx -= (vx > 0) - (vx < 0)
        vy -= (vy > 0) - (vy < 0)
        if vx == 0 and vy == 0:
            return True
        if not segment_ok(track, x, y, x + vx, y + vy):
            return False
        x += vx
        y += vy
    return True


def next_move(state):
    track = state["track"]
    me = state["you"]
    best = None
    for ax in (-1, 0, 1):
        for ay in (-1, 0, 1):
            vx, vy = me["vx"] + ax, me["vy"] + ay
            if abs(vx) > 2 or abs(vy) > 2:
                continue
            nx, ny = me["x"] + vx, me["y"] + vy
            if not segment_ok(track, me["x"], me["y"], nx, ny):
                continue
            if not can_stop(track, nx, ny, vx, vy):
                continue
            # TODO: qui scegli la candidata migliore (es. distanza dal
            # traguardo con una BFS sul tracciato). Per ora: la prima buona.
            if best is None:
                best = [ax, ay]
    return best or [0, 0]


def rispondi(state):
    if state.get("winner") is not None:
        return None  # gara finita
    if state.get("track") is None:
        return None  # annuncio di inizio gara
    return {"move": next_move(state)}

Consigli

  • Il cuore del gioco è la frenata: prima di accelerare chiediti sempre "da lì riesco ancora a fermarmi?". È il check can_stop qui sopra.
  • Per scegliere dove andare, precalcola una BFS della distanza dal traguardo (8 direzioni, il tracciato non cambia durante la gara) e minimizzala a ogni atterraggio.
  • Il tracciato arriva in ogni messaggio ma è sempre lo stesso per tutta la gara: se ricalcolare la BFS ogni volta è troppo per 2 secondi di RP2040, tienila leggera.
  • Per l'emulatore le solite note: CPython, riesecuzione a ogni messaggio.
