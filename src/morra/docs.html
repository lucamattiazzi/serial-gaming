Morra cinese — API del bot

Il gioco più semplice da arbitrare e il più subdolo da giocare: vince chi legge nei pattern dell'avversario.

Come funziona

Una partita dura 20 round: vince chi ne porta a casa di più (i pareggi di round non contano; la partita può finire pari). Le mosse sono simultanee: a ogni round l'arbitro interroga entrambi i giocatori nello stesso momento, e nessuno vede la scelta dell'altro prima di aver risposto. Trasporto solito: righe JSON su seriale USB (115200 baud, 8N1).

La busta comune

Ogni messaggio che ricevi contiene anche due campi di busta, omessi negli esempi di questa pagina per brevità: game (qui "morra") e match, un UUID che identifica la singola partita e resta identico dal primo all'ultimo messaggio. All'inizio di ogni partita arriva inoltre un annuncio con la sola busta e tutti i campi di gioco a null: ignoralo (la tua rispondi ritorna None). È grazie alla busta che il main.py «router» installato dal Laboratorio smista ogni messaggio al modulo del gioco giusto (bot_<gioco>.py) — è ciò che rende possibile la modalità torneo.

{"game": "morra", "match": "3f2b1c4e-…", …campi del gioco…}

Messaggio in ingresso: il round

{"round": 6, "rounds": 20,
 "history": [{"you": "rock", "opp": "paper"}, {"you": "scissors", "opp": "paper"}, ...],
 "score": {"you": 2, "opp": 3},
 "winner": null}
Campo Significato
round / rounds Il round corrente (da 1) e quanti ne verranno giocati in totale.
history Tutta la storia della partita, dal tuo punto di vista: per ogni round giocato, la tua mossa e quella avversaria. È qui che si vince: cerca i pattern dell'avversario. Arriva completa a ogni round, così il tuo bot può restare senza stato.
score Round vinti finora, dal tuo punto di vista.
winner null durante la partita; "you"/"opp"/"TIE" nella notifica finale (a cui non devi rispondere).

La tua risposta

{"move": "rock"}

Una di "rock", "paper", "scissors". Carta batte sasso, forbici battono carta, sasso batte forbici. Le regole:

  • Tempo massimo: 1 secondo per round. Nessuna risposta = perdi tutta la partita.
  • Mossa non valida (token diverso dai tre) = sconfitta immediata.
  • Disconnessione = sconfitta.

Esempio completo

import random

HANDS = ["rock", "paper", "scissors"]
BEATEN_BY = {"rock": "paper", "paper": "scissors", "scissors": "rock"}


def next_move(history):
    if len(history) < 3:
        return random.choice(HANDS)
    # contromossa alla mossa piu' frequente dell'avversario
    opp_moves = [r["opp"] for r in history]
    most_frequent = max(HANDS, key=opp_moves.count)
    return BEATEN_BY[most_frequent]


def rispondi(state):
    if state.get("winner") is not None:
        return None  # partita finita
    if state.get("round") is None:
        return None  # annuncio di inizio partita
    return {"move": next_move(state["history"])}

Consigli

  • Il puro caso non perde mai in valore atteso, ma nemmeno vince: contro bot deterministici serve riconoscere i pattern — frequenze, catene di Markov sulle ultime mosse, reazioni al risultato del round precedente ("chi vince ripete, chi perde cambia").
  • Occhio a essere prevedibile a tua volta: un pizzico di casualità ti protegge dai counter-counter.
  • Nella sfida al meglio di 5 ogni partita riparte da capo, storia compresa.
  • Per l'emulatore: le solite note (CPython, riesecuzione a ogni messaggio — la storia completa nel messaggio esiste apposta).
