Tron — API del bot

Due moto luminose, una griglia, scie che diventano muri. L'ultimo vivo vince.

Come funziona

Griglia 40×30. A ogni tick (~150 ms) entrambe le moto avanzano di una cella nella loro direzione corrente, e la cella che lasciano diventa un muro permanente. Muore chi finisce fuori griglia, su una scia (anche la propria) o in collisione frontale con l'avversario (in quel caso muoiono entrambi: pareggio).

Come nel pong, il gioco è in tempo reale: a ogni tick ricevi lo stato e puoi rispondere con una direzione, che resta attiva finché non ne mandi un'altra. Non sei obbligato a rispondere a ogni tick, ma un bot muto per più di 3 secondi perde. Trasporto solito: righe JSON su seriale USB.

La busta comune

Ogni messaggio che ricevi contiene anche due campi di busta, omessi negli esempi di questa pagina per brevità: game (qui "tron") e match, un UUID che identifica la singola partita e resta identico dal primo all'ultimo messaggio. All'inizio di ogni partita arriva inoltre un annuncio con la sola busta e tutti i campi di gioco a null: ignoralo, non rispondere. Grazie alla busta un solo main.py può giocare a tutti i giochi, scegliendo il gestore in base a state["game"] — è ciò che rende possibile la modalità torneo.

{"game": "tron", "match": "3f2b1c4e-…", …campi del gioco…}

Parti sempre da sinistra

Il tuo bot si vede sempre partire dal lato sinistro, diretto verso destra: per il giocatore di destra l'arbitro specchia il campo (coordinate x, righe della griglia e direzioni left/right), in entrambe le direzioni — anche i tuoi comandi. Il tuo codice non deve gestire il caso "parto da destra".

Messaggio in ingresso: lo stato

{"field": {"w": 40, "h": 30},
 "grid": ["........", "..YYY...", "....O...", ...],
 "you": {"x": 12, "y": 15, "dir": "right"},
 "opp": {"x": 27, "y": 14, "dir": "down"},
 "tick": 42,
 "winner": null}
Campo Significato
grid 30 stringhe da 40 caratteri (riga 0 in alto): . libero, Y la tua scia, O quella avversaria. Le teste sono incluse nelle scie.
you / opp Posizione della testa e direzione corrente ("up", "down", "left", "right"). y cresce verso il basso.
tick Numero del passo di gioco.
winner null durante il gioco; "you"/"opp"/"TIE" nella notifica finale.

La tua risposta

{"move": "up"}
  • Una di "up", "down", "left", "right". Qualsiasi altro valore = sconfitta immediata.
  • L'inversione a U è ammessa ma suicida: ti schianti sulla tua scia.
  • Silenzio massimo: 3 secondi, poi perdi.
  • Disconnessione a partita in corso = sconfitta.

Esempio completo

import sys
import json

DIRS = {"up": (0, -1), "down": (0, 1), "left": (-1, 0), "right": (1, 0)}


def is_free(state, x, y):
    w = state["field"]["w"]
    h = state["field"]["h"]
    if x < 0 or x >= w or y < 0 or y >= h:
        return False
    return state["grid"][y][x] == "."


def next_move(state):
    me = state["you"]
    # tira dritto finche' puoi, altrimenti svolta sulla prima via libera
    order = [me["dir"]] + [d for d in DIRS if d != me["dir"]]
    for d in order:
        dx, dy = DIRS[d]
        if is_free(state, me["x"] + dx, me["y"] + dy):
            return d
    return me["dir"]  # spacciato comunque


while True:
    line = sys.stdin.readline()
    try:
        state = json.loads(line)
    except ValueError:
        continue
    if state.get("winner") is not None:
        continue  # partita finita
    if state.get("grid") is None:
        continue  # annuncio di inizio partita
    print(json.dumps({"move": next_move(state)}))

Consigli

  • Sopravvivere ≠ vincere: la strategia vera è controllare più spazio dell'avversario. Un flood fill dalla tua testa (contro uno dalla sua) ti dice chi sta vincendo il territorio.
  • Gli stati arrivano ogni ~150 ms e la griglia pesa ~1.3 KB: tieni il parsing snello e rispondi in fretta, o giocherai su uno stato vecchio. Su MicroPython puoi scartare gli stati arretrati con select.poll su stdin.
  • Per l'emulatore le solite note: CPython, riesecuzione a ogni messaggio (~7 volte al secondo: inizializzazione di modulo leggerissima).
